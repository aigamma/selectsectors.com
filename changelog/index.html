v0.1.0 · 2026-05-18

Initial release. The site went from an empty directory to a feature-complete educational backtester over a multi-day autonomous build session. Everything in this list was implemented, tested, and verified end-to-end before v0.1.0 was tagged.

Backtester

  • Five strategies in the Rust crate at crates/backtest-core/: buy-and-hold, SMA crossover, momentum, RSI mean reversion (Wilder's recurrence), Donchian breakout. Compiled to WebAssembly via wasm-pack, loaded by the Netlify background function at request time.
  • No-lookahead enforcement at the engine layer: the apply_positions_to_bars function lags the position vector by one bar so individual strategy authors cannot accidentally cheat.
  • Result panel with six metrics: total return, CAGR, Sharpe (annualized), max drawdown, hit rate, bar count. Each label has a hover tooltip with a one-sentence definition.
  • Buy-and-hold benchmark equity curve overlaid on the equity chart, with a "vs buy-and-hold: +X%" summary line above the chart. Skipped when the user already picked buy-and-hold.

Cross-axis exploration

  • /compare/ runs all five strategies against one symbol in a single backtest with an overlay chart of all five equity curves and a Sharpe-ranked table.
  • /scan/ runs one strategy against all 23 symbols (SPX, SPY, 11 SPDR sectors, 11 anchor single names) in a single backtest with a Sharpe-ranked table. Symbol cells link back to the homepage with the strategy + symbol pre-filled.
  • Both consume one rate-limit slot regardless of internal backtest count.

Educational content

  • Six Rust curriculum lessons at /learn/: Why Rust, A tour of this site's Rust, Ownership and borrowing, Enums and dispatch, Error handling without thiserror, Rust to WebAssembly. Each lesson inlines code from crates/backtest-core/ via Vite's ?raw import, so the curriculum cannot drift from the source.
  • Five strategy explainer pages at /strategies/ with intuition, math, and the actual Rust source for each strategy. "Try it" links pre-fill the backtester form.
  • Four backtesting-philosophy primers at /philosophy/: overfitting, survivorship bias, lookahead bias, backtest vs live.
  • Three quiz categories at /quiz/: Rust basics, This site's Rust, Quant finance basics. 21 multiple-choice questions total with explanations and localStorage progress persistence.

SelectBot chatbot

  • Streaming chat via Server-Sent Events backed by the Anthropic SDK (claude-sonnet-4-6). Floating panel available on every page, conversation persisted to localStorage.
  • Focused system prompt with prompt caching: covers Rust (especially as this site uses it), the site itself, quant finance basics, and the philosophy of backtesting. Refuses investment advice and stock predictions; politely defers off-topic questions.
  • Per-IP rate limit independent of the backtester: 30 messages/hour and 100/day.

Tooling and infrastructure

  • 76 unit tests: 39 Rust (strategy parameter validation, metric correctness, no-lookahead lag) + 37 TypeScript (canonical JSON, rate-limit window math, strategy translation, HTML escape).
  • Three-job GitHub Actions CI workflow: cargo test, Vitest, full production build via wasm-pack + Vite.
  • Per-IP rate limiting with rolling-window resets (not fixed-clock, so no boundary-spike abuse).
  • Content-hash cache for backtest results: re-running the same backtest returns the cached blob instantly without consuming a rate-limit slot.
  • Structured data on every content page: WebSite + Organization on the homepage, Article on lessons + strategies + philosophy essays, BreadcrumbList on all nested pages.
  • Open Graph + Twitter Card meta tags on all 26 pages with a custom 1200x630 SVG OG image.
  • Accessibility: skip-to-content link, aria-labelled navigation, breadcrumbs with aria-current, semantic HTML.
What's next. Future versions will add more strategies (parameter-sweep view, custom strategies, more quant primitives), expanded curriculum (deeper Rust topics, a WASM internals lesson, a section on numerical methods), and operational polish (PNG OG rasterization for Twitter, performance audit, expanded daily_eod history once the Massive backfill runs). The pace of post-v0.1.0 work will be slower than the build session that produced v0.1.0.
