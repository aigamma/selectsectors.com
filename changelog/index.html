Changelog

What's new.

Reverse-chronological log of what shipped on each release. Versions follow semver; the site is currently v0.1.4 (a stable release, not a stable API since most of the API is internal).

v0.1.4 · 2026-05-18 (still same day)

Post-v0.1.3 stabilization pass with one real bug fix, a slate of error-path UX improvements, and a doubling of the regression-test surface.

  • Bug fix. STRATEGY_SPECS in src/strategy-specs.ts was missing the bollinger_bands entry that landed in v0.1.1. Users picking Bollinger Bands from the homepage or /scan/ select saw an empty param form and submit failed server-side with a serde "missing field" error. Now fixed; the per-strategy parity test catches future occurrences.
  • Error-path UX. Graceful failure rendering for failed /api/universe and /api/rate-status page-load fetches (previously stuck at "loading..." indefinitely). Rate-limit error messages now name the actual reset time ("resets in 23 min" or "in 5 hr 12 min") rather than telling users to look at the banner. The chat panel preemptively disables send + input when the chat rate limit is exhausted OR when ANTHROPIC_API_KEY is unset on the deploy. Shared URLs with unknown ?strategy= or ?symbol= values now show a non-blocking warning instead of silently no-opping.
  • Source-of-truth regression tests. Test count went 159 to 279 (+120). Nine new parity classes: vite-config-vs-HTML-pages, HTML-script-entries, env-vars- vs-.env.example, frontend STRATEGY_SPECS ↔ backend STRATEGY_DEFAULTS, strategy-specs ↔ explainer pages + catalog rows, quiz catalog (JSON ↔ entry ↔ page ↔ row + self-slug), philosophy + learn catalog rows, Rust Params field names ↔ toStrategyKind wire output, and chat-prompt strategy parity. All tests are discovery- based (readdirSync, Object.keys) so adding a new strategy / quiz / essay auto-extends the test suite without manual edits.
  • Polish. Curriculum + philosophy + quiz exits with "What's next" sections at the final lesson + final philosophy essay + every quiz completion. Time- until-reset formatter (in min / hr / day cascading) used site-wide. Hero copy + chat welcome prompts rebalanced. Architecture docs synced to the dual-store rate-limit layout. API docs 503 response shape fixed against the actual chat function. Strategy explainer fact-checks for Bollinger (Wilder/Bollinger author attribution), Turtle Trader (System 1 + System 2 entry rules), and momentum (Jegadeesh-Titman 1993 rather than "oldest documented factor").

v0.1.3 · 2026-05-18 (later same day)

Post-v0.1.2 polish pass focused on source-of-truth consolidation, automated drift detection, and the two reference pages that were missing for users who want to look up vocabulary or integrate against the API directly.

  • New /glossary/ page: alphabetical reference of every quant-finance term (annualized Sharpe, CAGR, drawdown, hit rate, lookahead bias, mean reversion, momentum, regime, RSI, Sharpe, SMA, survivorship bias, volatility) and every Rust term (borrow, cargo, crate, enum, lifetime, module, ownership, Result, serde, trait, WebAssembly, wasm-bindgen, wasm-pack) used across the site. 35 entries total, cross-linked back into the curriculum and philosophy pages where the term first appears in context.
  • New /api-docs/ page: developer- facing HTTP reference for the nine endpoints under /api/. Documents method + request body shape + response shapes for all status codes + rate-limit treatment for each endpoint. Includes the result-blob shape for single- backtest + /api/compare + /api/scan, the RateLimitInfo shape, and the strategy-catalog cheat sheet with default parameter values.
  • Centralized the 22-name universe roster (11 SPDR sectors + 11 anchor single names) in netlify/functions/_lib/universe-roster.mts. Previously duplicated across three files (universe.mts, scan-background.mts, chat-system-prompt.mts) and now imported from one canonical source.
  • Resolved a long-standing 23-vs-24 universe-count ambiguity: SPY is no longer included in scan results (its returns are near-identical to SPX and the row was visually redundant), so scan now operates on 23 symbols exactly matching the homepage form's pickable-options count.
  • Added a JSON-validity regression test that parses every <script type="application/ld+json"> structured-data block and every JSON example in /api-docs/. 64 new test cases. Catches the source-of- truth defects (trailing commas, mis-escaped quotes, accidental TypeScript-union notation) that earlier iterations had been fixing by hand.
  • Added a Rust-TS strategy parity test that parses the StrategyKind enum from mod.rs at test time and asserts the variants match STRATEGY_DEFAULTS in strategy.mts. Catches the drift where a strategy lands in Rust but the TS catalog doesn't know about it (or vice versa).
  • Added a universe-roster parity test that asserts the chat system prompt actually interpolates the ANCHORS list rather than carrying a stale hardcoded copy.
  • "Built in Rust. Learn it here." callout on the homepage surfaces /learn/ + /quiz/ + /glossary/ from the primary entry point so the Rust-teaching identity is visible without requiring users to discover it via the top-nav.
  • Test count grew from 117 (44 Rust + 73 TS) to 154 (44 Rust + 110 TS) over this release.

v0.1.0 · 2026-05-18

Initial release. The site went from an empty directory to a feature-complete educational backtester over a multi-day autonomous build session. Everything in this list was implemented, tested, and verified end-to-end before v0.1.0 was tagged.

Backtester

  • Five strategies in the Rust crate at crates/backtest-core/: buy-and-hold, SMA crossover, momentum, RSI mean reversion (Wilder's recurrence), Donchian breakout. Compiled to WebAssembly via wasm-pack, loaded by the Netlify background function at request time.
  • No-lookahead enforcement at the engine layer: the apply_positions_to_bars function lags the position vector by one bar so individual strategy authors cannot accidentally cheat.
  • Result panel with six metrics: total return, CAGR, Sharpe (annualized), max drawdown, hit rate, bar count. Each label has a hover tooltip with a one-sentence definition.
  • Buy-and-hold benchmark equity curve overlaid on the equity chart, with a "vs buy-and-hold: +X%" summary line above the chart. Skipped when the user already picked buy-and-hold.

Cross-axis exploration

  • /compare/ runs all five strategies against one symbol in a single backtest with an overlay chart of all five equity curves and a Sharpe-ranked table.
  • /scan/ runs one strategy against all 23 symbols (SPX, 11 SPDR sectors, 11 anchor single names) in a single backtest with a Sharpe-ranked table. Symbol cells link back to the homepage with the strategy + symbol pre-filled.
  • Both consume one rate-limit slot regardless of internal backtest count.

Educational content

  • Six Rust curriculum lessons at /learn/: Why Rust, A tour of this site's Rust, Ownership and borrowing, Enums and dispatch, Error handling without thiserror, Rust to WebAssembly. Each lesson inlines code from crates/backtest-core/ via Vite's ?raw import, so the curriculum cannot drift from the source.
  • Five strategy explainer pages at /strategies/ with intuition, math, and the actual Rust source for each strategy. "Try it" links pre-fill the backtester form.
  • Four backtesting-philosophy primers at /philosophy/: overfitting, survivorship bias, lookahead bias, backtest vs live.
  • Three quiz categories at /quiz/: Rust basics, This site's Rust, Quant finance basics. 21 multiple-choice questions total with explanations and localStorage progress persistence.

SelectBot chatbot

  • Streaming chat via Server-Sent Events backed by the Anthropic SDK (claude-sonnet-4-6). Floating panel available on every page, conversation persisted to localStorage.
  • Focused system prompt with prompt caching: covers Rust (especially as this site uses it), the site itself, quant finance basics, and the philosophy of backtesting. Refuses investment advice and stock predictions; politely defers off-topic questions.
  • Per-IP rate limit independent of the backtester: 30 messages/hour and 100/day.

Tooling and infrastructure

  • 76 unit tests: 39 Rust (strategy parameter validation, metric correctness, no-lookahead lag) + 37 TypeScript (canonical JSON, rate-limit window math, strategy translation, HTML escape).
  • Three-job GitHub Actions CI workflow: cargo test, Vitest, full production build via wasm-pack + Vite.
  • Per-IP rate limiting with rolling-window resets (not fixed-clock, so no boundary-spike abuse).
  • Content-hash cache for backtest results: re-running the same backtest returns the cached blob instantly without consuming a rate-limit slot.
  • Structured data on every content page: WebSite + Organization on the homepage, Article on lessons + strategies + philosophy essays, BreadcrumbList on all nested pages.
  • Open Graph + Twitter Card meta tags on all 26 pages with a custom 1200x630 SVG OG image.
  • Accessibility: skip-to-content link, aria-labelled navigation, breadcrumbs with aria-current, semantic HTML.
What's next. Future versions will add more strategies (parameter-sweep view, custom strategies, more quant primitives), expanded curriculum (deeper Rust topics, a WASM internals lesson, a section on numerical methods), and operational polish (PNG OG rasterization for Twitter, performance audit, expanded daily_eod history once the Massive backfill runs). The pace of post-v0.1.0 work will be slower than the build session that produced v0.1.0.
