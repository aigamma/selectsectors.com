Glossary.

Alphabetical reference for every domain-specific term used across the site. Quant-finance terms first, then Rust + WASM terms. If a term you need isn't here, ask SelectBot (bottom right) and we'll add it on the next deploy.

Quant finance

Annualized Sharpe
The Sharpe ratio computed on daily-bar returns and scaled by sqrt(252) to express it on an annual basis. The 252 comes from US trading days per year. See /quiz/quant-finance/.
Bar
One row of OHLC + volume data for one symbol on one trading day. The smallest unit of input to a backtest.
Benchmark
A reference performance series against which a strategy is compared. This site uses buy and hold as the universal benchmark, overlaid on every backtest's equity chart.
CAGR
Compound Annual Growth Rate. (1 + total_return)^(1/years) − 1. The honest way to talk about multi-year returns because it accounts for compounding.
Daily bar
A bar covering one trading day's open, high, low, close, and volume. The smallest time-granularity this site supports.
Drawdown
The decline from a running peak in an equity curve. Max drawdown is the worst such decline expressed as a positive fraction. See /philosophy/.
EOD
End-of-day. As in "EOD bar": the OHLC + volume snapshot of one trading day, finalized after the market close. This site is EOD-only.
Equity curve
The cumulative product of (1 + r_t) over a return series, normalized to start at 1.0. The chart on a backtest result panel is the equity curve.
Hit rate
Fraction of daily realized returns that are strictly positive. Differs in shape between trend-following strategies (lower hit rate, larger winners) and mean-reversion (higher hit rate, smaller winners).
Lookahead bias
Using a value in a signal computation that wouldn't have been known at the time. The most common version is using close[i] to set position[i]. This site enforces no- lookahead at the engine layer via apply_positions_to_bars. See /philosophy/lookahead-bias/.
Mean reversion
The tendency of prices to return toward a longer-term average. Strategies that exploit this fade extreme moves (RSI mean reversion, Bollinger Bands).
Momentum
The tendency of recent winners to keep winning (and losers to keep losing). Strategies that exploit this go long when prices are above their N-bar lookback.
OHLC
Open, High, Low, Close. The four price fields on a daily bar. SPX rows have open == high == low == close in this site's database because the upstream table is close-only; equity and ETF rows carry full OHLC.
Overfitting
Tuning a strategy's parameters to maximize backtest performance on a specific historical window. The result generalizes poorly out-of-sample. See /philosophy/overfitting/.
Regime
A stretch of market behavior with stable statistical character (trending vs sideways, high-vol vs low-vol, etc). Strategy performance is regime-dependent. See /philosophy/regimes/.
RSI (Relative Strength Index)
A 0-100 oscillator computed from the rolling balance of up-day vs down-day gains. Wilder's recurrence is the convention. Below 30 = oversold; above 70 = overbought.
Sharpe ratio
Mean return divided by standard deviation of returns. Scale-invariant (multiplying every return by k leaves Sharpe unchanged). Annualized by multiplying by sqrt(252) for daily bars.
SMA (Simple Moving Average)
The arithmetic mean of the last N bars' closes. Often paired with a faster SMA to form an SMA crossover signal.
Survivorship bias
Backtesting on the universe of names that survive to the present, ignoring the names that were once in the list but rotated out. See /philosophy/survivorship-bias/.
Total return
Cumulative percent return from first bar to last bar of the strategy's equity curve. Less useful than CAGR for multi-year comparisons.
Trend following
Strategies that bet prices will continue in the direction they're moving. Examples in this site's library: SMA crossover, momentum, Donchian breakout.
Volatility
Standard deviation of returns over a window, often annualized by sqrt(252). High vol = stressful conditions (typically >25% annualized); low vol = calm (typically <12%).

Rust and WebAssembly

Borrow
A reference to a value that does not transfer ownership. `&T` is a shared (immutable) borrow; `&mut T` is a unique (mutable) borrow. The borrow checker enforces that no value has both kinds of borrow simultaneously.
Cargo
Rust's package manager and build tool. `cargo test`, `cargo build`, `cargo install` are the commands you'll use most often.
Crate
The unit of compilation in Rust. A library crate produces an .rlib; a binary crate produces an executable. This site's `crates/backtest-core/` is a library crate compiled to WASM.
Enum (tagged union)
A Rust type that can be one of several variants, each potentially with its own payload. Pattern matching on an enum is exhaustive: the compiler enforces that every variant is handled. See /learn/enums-and-dispatch/.
Lifetime
A compile-time annotation that tracks how long a reference is valid. Usually inferred; explicit annotations like `'a` show up when the compiler can't tell from the signature alone.
Module
A unit of organization in Rust. `pub mod name;` declares a module that lives in `name.rs` or `name/mod.rs`. See /learn/this-sites-rust/.
Move semantics
Assigning a non-Copy value transfers ownership; the original binding can no longer be used. The compiler rejects use-after-move at compile time.
Ownership
Rust's core memory-safety system. Each value has a single owner; when the owner goes out of scope, the value is dropped. No garbage collector. See /learn/ownership/.
Result<T, E>
Rust's error type. Either `Ok(T)` or `Err(E)`. The `?` operator propagates an Err up the call stack; the compiler enforces handling. See /learn/error-handling/.
serde
Rust's serialization framework. The `#[derive(Serialize, Deserialize)]` macros generate the boilerplate for converting Rust structs to/from JSON (or other formats).
Trait
Like an interface in other languages: a collection of method signatures that types can implement. Generic bounds like `T: Display` say "T must implement Display".
WebAssembly (WASM)
A portable binary format for code that runs in a sandbox in browsers and Node.js. Rust compiles to WASM; the resulting binary runs at near-native speed.
wasm-bindgen
The Rust crate that generates the JS-WASM glue: a Rust trampoline that takes primitive arguments + a JS shim that marshals JS values into those primitives. The `#[wasm_bindgen]` attribute is the marker.
wasm-pack
The CLI tool that builds a Rust crate to WASM + emits the JS glue + writes a package.json wrapper. Three targets: nodejs, web, bundler.
