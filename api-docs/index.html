Reference

HTTP API reference.

The selectsectors.com backend is nine Netlify Functions rooted at /api/. Most are reads (GET) for status and roster data; three are writes (POST) that consume rate- limit slots and dispatch a background WASM run. This page is the single source of truth for what each endpoint accepts and returns. The canonical implementation lives at netlify/functions/.

Conventions

  • All endpoints accept and return application/json unless noted.
  • Origin is https://selectsectors.com. The /api/* prefix is rewritten to /.netlify/functions/* by netlify.toml.
  • Rate limit is per-IP, two rolling windows: backtest at 2/hour and 5/day; chat at 30/hour and 100/day.
  • Cache hits on /api/backtest, /api/compare, and /api/scan return { status: "ready", cached: true, result } immediately.
  • Cache MISSES still consume one rate-limit slot, then dispatch to a Netlify Background Function (15-min wall clock) and return 202 Accepted with a content hash; poll /api/result?hash=<hash> for completion.

Read endpoints (no rate limit)

GET /api/health

Operational health check. Reports the deployed version, commit, deploy ID, the catalog of WASM strategies, and the configured rate-limit caps for both backtest and chat.

{
  "status": "ok",
  "site": "selectsectors.com",
  "version": "0.1.2",
  "commit": "<git sha>",
  "deployId": "<netlify deploy id>",
  "timestamp": "2026-05-18T05:30:00.000Z",
  "strategies": [
    "buy_and_hold",
    "sma_crossover",
    "momentum",
    "rsi_mean_reversion",
    "breakout",
    "bollinger_bands"
  ],
  "rateLimits": {
    "backtest": { "hour": 2, "day": 5 },
    "chat":     { "hour": 30, "day": 100 }
  }
}

GET /api/universe

The 22-name roster from the upstream options-volume-roster: eleven SPDR sector ETFs plus eleven anchor single names. SPX is added client-side because it's the index (always present in the catalog) rather than a rotating universe member; SPY exists in the upstream Supabase table but is intentionally excluded from the public-site universe because its returns are near-identical to SPX.

{
  "sectors": ["XLB", "XLC", "XLE", "XLF", "XLI", "XLK",
              "XLP", "XLU", "XLV", "XLY", "XLRE"],
  "anchors": ["NVDA", "TSLA", "AAPL", "AMD", "AMZN", "META",
              "MSFT", "GOOGL", "PLTR", "COIN", "SMCI"],
  "note":    "Eleven SPDR sector ETFs plus eleven anchor single names..."
}

GET /api/rate-status

Current backtest rate-limit counters for the caller's IP, without consuming a slot. Used by the homepage to render the "you have N left" banner on page load.

{
  "caps": { "hour": 2, "day": 5 },
  "hourly": {
    "limit": 2, "used": 0, "remaining": 2,
    "resetAt": 1747560000000
  },
  "daily": {
    "limit": 5, "used": 0, "remaining": 5,
    "resetAt": 1747646400000
  }
}

resetAt is a Unix epoch milliseconds value, not a string. The window is rolling, not fixed-clock; each window resets exactly 60 minutes (or 24 hours) after the first request inside it.

GET /api/chat-status

Current chat rate-limit counters. Same shape as rate-status but with the chat caps (30/hour, 100/day) and the chat counter store. If ANTHROPIC_API_KEY is unset on the Netlify project, this endpoint still returns the counters; the failure mode shows up at /api/chat send time as a 503.

{
  "caps": { "hour": 30, "day": 100 },
  "hourly": { "limit": 30, "used": 0, "remaining": 30, "resetAt": <ms> },
  "daily":  { "limit": 100, "used": 0, "remaining": 100, "resetAt": <ms> }
}

Write endpoints (rate-limited)

POST /api/backtest

Run one strategy on one symbol for a date range. Synchronous dispatcher: computes the content hash, short- circuits on cache hit, otherwise consumes one rate-limit slot and fires the background function.

Request

{
  "symbol": "SPX",
  "strategy": {
    "name":   "sma_crossover",
    "params": { "fast": 20, "slow": 50 }
  },
  "dateRange": { "start": "2024-01-01", "end": "2025-12-31" }
}

Response 200 (cache hit)

{
  "status": "ready",
  "hash":   "<sha256 hex, 64 chars>",
  "cached": true,
  "result": { /* see Result shape below */ }
}

Response 202 (queued)

{
  "status": "queued",
  "hash":   "<sha256 hex>",
  "rateLimits": { /* RateLimitInfo */ }
}

Response 429 (rate-limited)

{
  "error":  "rate limit exceeded",
  "reason": "hour-exceeded",
  "rateLimits": { /* RateLimitInfo with current counters */ }
}

reason is either "hour-exceeded" or "day-exceeded" depending on which window bit first.

The Retry-After response header is set in seconds to the time until the relevant window resets.

POST /api/compare

Run all six strategies on one symbol over one date range using each strategy's default parameters. Consumes one rate-limit slot regardless of the internal backtest count. The hash input is tagged with "mode": "compare-all" so the cache key cannot collide with a single-strategy run on the same symbol + date range.

Request

{
  "symbol": "XLE",
  "dateRange": { "start": "2024-01-01", "end": "2025-12-31" }
}

See the Compare result shape section below for the full result blob structure.

POST /api/scan

Run one strategy across all 23 universe symbols for one date range. Consumes one rate-limit slot. Internally fans out to one Supabase query with an IN clause covering all 23 symbols, then runs the WASM strategy 23 times against the partitioned bars. Hash input is tagged with "mode": "scan-all".

Request

{
  "strategy": {
    "name":   "momentum",
    "params": { "lookback": 60 }
  },
  "dateRange": { "start": "2024-01-01", "end": "2025-12-31" }
}

See the Scan result shape section below for the full result blob structure.

POST /api/chat

SelectBot. Server-Sent Events streaming response from claude-sonnet-4-6 via the Anthropic SDK. Cached system prompt covers Rust, this site, quant finance basics, and backtesting philosophy. Consumes one chat rate-limit slot per send.

Request

{
  "messages": [
    { "role": "user", "content": "What's Wilder's RSI recurrence?" }
  ]
}

Response 200 (SSE stream)

Content-Type text/event-stream. The body forwards Anthropic's SDK streaming events verbatim; clients should concatenate content_block_delta events' delta.text fields to assemble the reply.

Response 503 (no API key)

{
  "error": "chat_unavailable",
  "reason": "ANTHROPIC_API_KEY not set on the Netlify project"
}

Poll endpoint (no rate limit)

GET /api/result?hash=<hex>

Fetch a backtest, compare, or scan result by content hash. hash must be 64 lowercase hex chars; any other shape returns 400. Returns 202 while the background function is still running; 200 once it's complete.

Response 202 (pending)

{ "status": "pending", "hash": "<hex>" }

Response 200 (ready)

{
  "status": "ready",
  "hash":   "<hex>",
  "result": { /* the appropriate result shape from above */ }
}

Response 400 (malformed hash)

{
  "error": "missing or malformed hash; expected a 64-char hex sha256"
}

Result shapes

Single backtest (/api/backtest)

What backtest-background.mts writes into the result blob and what /api/result?hash= returns under result. The original request body is echoed back under inputs; computed metrics live at the top level; the equity curve is an array of objects (one per bar) carrying the trading date plus the per-bar return and the cumulative equity value.

{
  "inputs": {
    "symbol":    "SPX",
    "strategy":  { "name": "sma_crossover", "params": { "fast": 20, "slow": 50 } },
    "dateRange": { "start": "2024-01-01", "end": "2025-12-31" }
  },
  "bars":             504,
  "firstDate":        "2024-01-02",
  "lastDate":         "2025-12-30",
  "totalReturn":      0.187,
  "annualizedReturn": 0.093,
  "sharpe":           1.04,
  "maxDrawdown":      0.072,
  "hitRate":          0.542,
  "equityCurve": [
    { "date": "2024-01-02", "ret":  0.000, "equity": 1.000 },
    { "date": "2024-01-03", "ret":  0.002, "equity": 1.002 }
    /* ... one entry per bar ... */
  ],
  "benchmark": {
    "name":             "buy_and_hold",
    "totalReturn":      0.211,
    "annualizedReturn": 0.108,
    "sharpe":           1.18,
    "maxDrawdown":      0.058,
    "equityCurve": [ /* same shape, one entry per bar */ ]
  },
  "note":       "WASM ... over 504 bars from 2024-01-02 to 2025-12-30",
  "computedAt": "2026-05-18T05:30:00.000Z",
  "computeMs":  847
}

benchmark is null when the user's chosen strategy is itself buy_and_hold (the result and benchmark would be byte-identical).

Compare (/api/compare)

What compare-background.mts writes. One entry per strategy in the strategies array; already sorted by Sharpe descending. Per-strategy failures carry an error field instead of metrics.

{
  "inputs":    { /* echoed request */ },
  "mode":      "compare-all",
  "bars":      504,
  "firstDate": "2024-01-02",
  "lastDate":  "2025-12-30",
  "strategies": [
    {
      "name":             "momentum",
      "totalReturn":      0.234,
      "annualizedReturn": 0.118,
      "sharpe":           1.41,
      "maxDrawdown":      0.083,
      "hitRate":          0.534,
      "equityCurve": [ /* per-bar */ ]
    },
    /* ... five more, or one of: */
    { "name": "rsi_mean_reversion", "error": "NotEnoughBars: ..." }
  ],
  "computedAt": "...",
  "computeMs":  4123
}

Scan (/api/scan)

What scan-background.mts writes. One entry per symbol in the symbols array; sorted by Sharpe descending. Equity curves are deliberately omitted per-symbol to keep the blob compact (~5-12K points across 23 symbols would bloat both the result blob and the frontend payload). Use /api/backtest for the full equity curve on any one symbol.

{
  "inputs":    { /* echoed request */ },
  "mode":      "scan-all",
  "strategy":  "momentum",
  "dateRange": { "start": "2024-01-01", "end": "2025-12-31" },
  "symbols": [
    {
      "symbol":           "NVDA",
      "category":         "anchor",
      "bars":             504,
      "firstDate":        "2024-01-02",
      "lastDate":         "2025-12-30",
      "totalReturn":      0.847,
      "annualizedReturn": 0.382,
      "sharpe":           1.83,
      "maxDrawdown":      0.214,
      "hitRate":          0.521
    },
    /* ... 22 more, or one of: */
    { "symbol": "XYZ", "category": "sector",
      "error":  "not enough bars (0) in 2024-01-01..2025-12-31" }
  ],
  "computedAt": "...",
  "computeMs":  3812
}

category is one of "index" (SPX), "sector" (the 11 SPDR ETFs), or "anchor" (the 11 single names).

Error result blob

When a background function fails before producing metrics (Supabase env missing, fetch bars failed, WASM crashed, bad strategy params), it writes an error blob rather than nothing, so /api/result can distinguish a still-running run from a failed one:

{
  "error":      "fetch bars failed: ...",
  "inputs":     { /* echoed request */ },
  "computedAt": "..."
}

RateLimitInfo shape

Returned by /api/rate-status, /api/chat-status, and embedded in the 202 and 429 responses from /api/backtest + /api/compare + /api/scan. Two windows, each with limit + used + remaining + resetAt (ms Unix epoch).

{
  "hourly": { "limit": <n>, "used": <n>, "remaining": <n>, "resetAt": <ms> },
  "daily":  { "limit": <n>, "used": <n>, "remaining": <n>, "resetAt": <ms> }
}

Strategy catalog

Every strategy's name tag plus its parameter names. Defaults match the values exported from netlify/functions/_lib/strategy.mts and consumed by the compare endpoint.

buy_and_hold        (no params)
sma_crossover       fast: number, slow: number               (defaults: 20, 50)
momentum            lookback: number                          (default: 60)
rsi_mean_reversion  period: number, oversold: number, overbought: number
                                                              (defaults: 14, 30, 70)
breakout            lookback: number                          (default: 20)
bollinger_bands     period: number, k: number                 (defaults: 20, 2.0)

Status codes

200 OK
Read endpoints succeeded, or a cached backtest result was returned, or a polled result is ready.
202 Accepted
Write endpoints when the background function was dispatched; poll /api/result. Also returned by /api/result itself when the requested hash is still pending.
400 Bad Request
Malformed JSON body, missing required field, or (for /api/result) a hash that does not match the 64-char hex sha256 pattern.
405 Method Not Allowed
The write endpoints (/backtest, /compare, /scan) only accept POST.
429 Too Many Requests
Rate-limit exhausted. Response body carries reset timestamps; Retry-After header set in seconds.
503 Service Unavailable
/api/chat only, when ANTHROPIC_API_KEY is unset on the Netlify project.
Try it. curl https://selectsectors.com/api/health returns the live catalog. The version, deployed commit, and strategy list let you sanity-check that the deploy you're hitting is the deploy you expect.
