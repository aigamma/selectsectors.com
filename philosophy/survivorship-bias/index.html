Survivorship bias.

Backtesting on today's universe of names misses the names that have rotated out. The names you can backtest today are the ones still around to backtest; the ones that delisted, got acquired, or fell off the relevant ranking are silently excluded. The universe selection is as important as the strategy itself.

The most famous example

Back-test "buy the top eleven by options volume" using the 2026 list (NVDA, TSLA, AAPL, AMD, AMZN, META, MSFT, GOOGL, PLTR, COIN, SMCI) on data from 2018. You'll get a phenomenal result, because those eleven names are the ones that survived to be the top-by-options-volume in 2026. The equivalent 2018 list would have included names like GE, Bank of America, and a different mix entirely. Many of those have rerated lower since.

This is the same trap as picking the top eleven mutual funds of 2026 and back-testing their pre-2026 performance. The selection is post-hoc; the result is misleading.

How to think about it

A backtest is only honest if the universe at each historical point in time matches the universe you would have actually held at that point. Two ways to satisfy this:

  1. Hold a fixed universe across the test window. (SPY is SPY; SPX is SPX. The index composition changes underneath but the ticker itself stays constant.)
  2. Use a point-in-time universe: at each rebalance date, use the universe that would have been visible at that date, not the current one. This requires a historical-universe data source, which is expensive and rare.

This site's universe is the eleven SPDR sector ETFs, the eleven anchor single names, plus SPX and SPY. The sector ETFs and SPX/SPY are stable; the eleven anchor names are where survivorship bias creeps in if you forget to think about it. The anchor list rotates as new names dominate options volume; the current list reflects 2026 rankings.

What the symptom looks like

A strategy that looks great on today's top-N names and poor on yesterday's top-N names. The gap between the two backtests is roughly the survivorship-bias-adjusted true expected return: the rest is the universe-selection alpha you would not have captured in real time.

Index funds dodge this because the index itself does the rebalancing for you, with the names that drop out accounted for in the index methodology. Single-stock strategies that pick "top names" by any current metric do not get this for free.

Practical rule. If your backtest uses a universe you selected with knowledge of recent performance, the result is overstated by an amount you can't easily quantify. Either test on a fixed universe-by-ticker (SPY, sector ETFs) or be honest about the bias when you cite the result.
